Frequently Asked Questions

For additional help, check UBIT Alerts and contact the UBIT Help Center.

Results and UB connectivity

{% call faq_item('faq-what-you-see', 'fa-circle-info', 'primary', 'What can this tool show?') %}

Every visitor can see the IP address visible to this application and whether IPv4 and IPv6 requests succeed. The page may also show internet provider, approximate geolocation and DNS information.

When the address is recognized as a UB network or UB VPN address and the deployment has the necessary integrations, additional network, VLAN, DHCP, IP address management, wireless, device and UB location details may appear. Missing details can mean the integration has no record; it does not necessarily mean the connection is broken.

{% endcall %} {% call faq_item('faq-wifi', 'fa-wifi', 'primary', 'Which Wi-Fi network should I use at UB?') %}

eduroam is UB's official secure Wi-Fi network for students, faculty and staff. UB_Connect is used by visitors and as the on-campus starting point for eduroam setup. UB_Devices is for devices such as some TVs, printers and gaming consoles that cannot use enterprise authentication.

Set up eduroam and review UB Wi-Fi options.

{% endcall %} {% call faq_item('faq-off-campus-result', 'fa-house', 'primary', 'Why am I seeing an off-campus result while physically at UB?') %}
  • You may be connected to guest Wi-Fi rather than eduroam.
  • Your device may be using a personal hotspot or cellular connection.
  • A VPN from another organization may be routing the request elsewhere.
  • The device may be on a specialized UB network range that is not yet configured in this deployment.

Connect to eduroam or a UB wired network, disable unrelated VPN software and run the test again.

{% endcall %} {% call faq_item('faq-vpn', 'fa-shield-halved', 'primary', 'How should I interpret a UB VPN result?') %}

UB recommends Cisco Secure Client to access protected UB services from off campus. Some VPN configurations route only traffic intended for protected UB services. Because of that, a general public IP check may continue to show your home or mobile provider even while selected UB services use the VPN.

Use the result together with the service you are trying to reach. See UB VPN guidance for current setup instructions.

{% endcall %} {% call faq_item('faq-different-ip', 'fa-arrow-right-arrow-left', 'primary', 'Why does this site show a different IP address than another lookup site?') %}

Different sites can be reached over different protocols and routes. One site may use IPv6 while another uses IPv4; a VPN, proxy, secure web gateway, cellular relay or split-routing policy can also present different public addresses. Both results can be valid for their respective destinations.

{% endcall %}

Network and DNS

{% call faq_item('faq-campus', 'fa-building', 'success', 'What does “UB network” mean on this site?') %}

It means an address range configured by the deployment team as belonging to University at Buffalo network infrastructure, including recognized campus and remote UB locations. UB has North, South and Downtown campuses, and also operates at other locations. See the official UB campus maps.

{% endcall %} {% call faq_item('faq-nac', 'fa-lock', 'success', 'What is Network Access Control?') %}

Network Access Control is a general term for systems that identify devices and apply connection policy. If this deployment is integrated with UB network-management systems, it may show connection information available for the current address. The exact fields depend on the data source and permissions configured by the deployment team.

{% endcall %} {% call faq_item('faq-dns-provider', 'fa-magnifying-glass', 'success', 'What does the DNS provider check show?') %}

The test estimates which resolver answered a browser-generated DNS query. It is similar to a lightweight DNS leak check, but the result should be treated as diagnostic evidence rather than a definitive security verdict.

Encrypted DNS, browser settings, UB VPN routing, cellular privacy relays, enterprise security products and the network's resolver architecture can all affect the provider name. Do not assume that one specific company name is always required at UB.

The provider label may be derived from third-party IP and DNS data and can be incomplete or outdated.

{% endcall %} {% call faq_item('faq-dns-security', 'fa-filter-circle-dollar', 'success', 'What does the DNS security test mean?') %}

This feature only has meaning when the deployment administrator has configured a test domain and documented the expected behavior. A passed or failed result can be affected by caching, browser security, encrypted DNS and network filtering. Do not describe it as an official UB security control unless UBIT has approved the configuration and wording.

{% endcall %}

Privacy and location

{% call faq_item('faq-privacy', 'fa-eye-slash', 'warning', 'Is my IP address stored or tracked?') %}

The application's metrics feature is designed to store aggregate counts and classifications rather than raw client IP addresses.

That does not cover every system in the hosting path. Web-server, reverse-proxy, load-balancer, security and operational logs may record network identifiers according to their configuration and applicable UB policy. The production owner should complete a privacy and logging review before launch.

{% endcall %} {% call faq_item('faq-external-service', 'fa-arrow-up-right-from-square', 'warning', 'Does this site contact external services?') %}

Depending on configuration, the browser may contact:

  • ip-api.com for IP, provider or DNS-related data.
  • ipify.org to compare the IPv4 address visible to a public internet service.
  • A configured DNS test domain.
  • Connectivity targets listed on the Connectivity page.
  • Fast.com only when the user opens the external speed-test link.

Each external provider can see the public IP address used to contact it. The deployment owner should confirm the enabled services, privacy notice and third-party review requirements before production use.

{% endcall %} {% call faq_item('faq-location-accuracy', 'fa-location-dot', 'warning', 'Why does my IP location look inaccurate?') %}

IP geolocation is an estimate based on address-registration and provider data, not GPS. It may identify Buffalo, Amherst, another nearby city or a provider facility rather than your physical location. Campus integrations can provide a more specific building only when the deployment has a reliable authorized data source.

{% endcall %} {% call faq_item('faq-location-update', 'fa-pen-to-square', 'warning', 'Can inaccurate public geolocation data be corrected?') %}

The public geolocation provider used by the application may accept correction requests. For IP-API, see its data correction page. UB-managed address data should be corrected through the appropriate UB network or IP address management process instead.

{% endcall %}

Reports, speed and API

{% call faq_item('faq-report', 'fa-file-arrow-down', 'info', 'How do I download a support report?') %}

The Report button creates a print-ready summary after the home-page diagnostics finish loading. Use your browser's Print command and select Save as PDF.

Attach the result to a request at support.buffalo.edu. Review the report first and do not include passwords, authentication tokens or restricted data.

{% endcall %} {% call faq_item('faq-speedtest', 'fa-gauge-high', 'info', 'How does the speed-test page work?') %}

The page opens Fast.com, a third-party service referenced in UBIT connectivity guidance. The test measures performance to the provider's infrastructure, not specifically to a UB campus server.

Results vary with Wi-Fi signal, other traffic, VPN routing, device performance and network congestion. Compare multiple tests and note whether you were on eduroam, wired Ethernet, a home network or cellular data.

{% endcall %} {% call faq_item('faq-isp-org', 'fa-sitemap', 'info', 'What is the difference between Internet provider and Organization?') %}

Internet provider generally names the company providing or announcing the connection. Organization generally identifies the entity associated with the specific IP block.

For a campus address, the organization may appear as the University at Buffalo, SUNY, a network partner or another registry label. These fields come from public IP data and are not authoritative UB asset records.

{% endcall %} {% call faq_item('faq-query', 'fa-ban', 'info', 'Can I query information about a different IP address?') %}

No. The application is designed to report information about the client connection making the request, not arbitrary third-party addresses.
